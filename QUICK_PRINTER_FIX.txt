║              QUICK PRINTER FIX - POS-58                         ║
╚══════════════════════════════════════════════════════════════════╝

PROBLEMA: POS system working pero hindi nag-print ang POS-58

┌──────────────────────────────────────────────────────────────────┐
│  STEP 1: CHECK PRINTER IN WINDOWS                               │
└──────────────────────────────────────────────────────────────────┘

1. Press Windows + I (Settings)
2. Go to: Devices → Printers & scanners
3. Find: POS-58
4. Check status:
   ✅ Ready (green) = Good
   ❌ Offline (gray) = Problem
   ❌ Error (red) = Problem

┌──────────────────────────────────────────────────────────────────┐
│  STEP 2: TEST PRINT FROM WINDOWS                                │
└──────────────────────────────────────────────────────────────────┘

1. Right-click sa POS-58 printer
2. Click "Manage"
3. Click "Print test page"
4. Check kung nag-print

┌──────────────────────────────────────────────────────────────────┐
│  STEP 3: CHECK PRINTER SHARING                                  │
└──────────────────────────────────────────────────────────────────┘

1. Right-click sa POS-58 printer
2. Click "Printer properties"
3. Go to "Sharing" tab
4. Check "Share this printer"
5. Set share name: POS-58
6. Click "Apply" and "OK"

┌──────────────────────────────────────────────────────────────────┐
│  STEP 4: RESTART XAMPP AS ADMINISTRATOR                         │
└──────────────────────────────────────────────────────────────────┘

1. Close XAMPP completely
2. Right-click: RUN_XAMPP_AS_ADMIN.bat
3. Click "Run as administrator"
4. Start Apache
5. Test ulit sa POS

┌──────────────────────────────────────────────────────────────────┐
│  STEP 5: CHECK PRINTER NAME                                     │
└──────────────────────────────────────────────────────────────────┘

Kung iba ang printer name mo:

1. Open: Api/print-receipt-fixed-width.php
2. Find line 152: $printerName = 'POS-58';
3. Change to exact printer name mo
4. Save file
5. Test ulit

┌──────────────────────────────────────────────────────────────────┐
│  COMMON ISSUES & SOLUTIONS                                      │
└──────────────────────────────────────────────────────────────────┘

ISSUE 1: Printer shows "Offline"
SOLUTION:
  - Check USB connection
  - Turn printer off and on
  - Restart printer service

ISSUE 2: "Access is denied"
SOLUTION:
  - Run XAMPP as Administrator
  - Share printer on network
  - Check printer permissions

ISSUE 3: "The system cannot find the file"
SOLUTION:
  - Check exact printer name
  - Verify printer is installed
  - Check printer sharing

ISSUE 4: No error but no print
SOLUTION:
  - Check print queue for stuck jobs
  - Clear print queue
  - Restart print spooler service

┌──────────────────────────────────────────────────────────────────┐
│  QUICK TEST                                                      │
└──────────────────────────────────────────────────────────────────┘

1. Open: http://localhost/caps2e2/SIMPLE_PRINT_TEST.html
2. Click "Test Print"
3. If success = Printer working, POS issue
4. If failed = Printer hardware issue
