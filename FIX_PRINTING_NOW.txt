║              AYUSIN ANG PRINTING - SIMPLE STEPS                  ║
╚══════════════════════════════════════════════════════════════════╝

PROBLEMA: Nag-checkout na pero hindi nag-print ang receipt

┌──────────────────────────────────────────────────────────────────┐
│  STEP 1: CHECK XAMPP (IMPORTANTE!)                              │
└──────────────────────────────────────────────────────────────────┘

1. CLOSE ang XAMPP kung naka-open
2. RIGHT-CLICK sa RUN_XAMPP_AS_ADMIN.bat
3. Click "Run as administrator"
4. Sa XAMPP Control Panel, click START sa Apache
5. Dapat GREEN ang Apache

⚠️ KAILANGAN AS ADMINISTRATOR para gumana ang printing!

┌──────────────────────────────────────────────────────────────────┐
│  STEP 2: CHECK PRINTER                                          │
└──────────────────────────────────────────────────────────────────┘

1. Press Windows + I (Settings)
2. Go to: Devices → Printers & scanners
3. Find: POS-58
4. Check:
   ✓ Status: Ready (hindi "Offline")
   ✓ May papel
   ✓ Naka-on
5. Click "Manage" → "Print test page"
6. Dapat mag-print ✓

┌──────────────────────────────────────────────────────────────────┐
│  STEP 3: TEST PRINT FROM WEB                                    │
└──────────────────────────────────────────────────────────────────┘

1. Open browser:
   http://localhost/caps2e2/test_print_direct.html

2. Click "🖨️ Test Print Receipt"

3. Tingnan ang result:
   ✓ SUCCESS = Gumana! Mag-print na sa POS
   ✗ ERROR = May problema pa

┌──────────────────────────────────────────────────────────────────┐
│  STEP 4: TEST SA POS SYSTEM                                     │
└──────────────────────────────────────────────────────────────────┘

Kung gumana ang Step 3:

1. Open POS: http://localhost:3000/POS_convenience
2. Add items to cart
3. Complete checkout
4. Dapat automatic mag-print! ✓

┌──────────────────────────────────────────────────────────────────┐
│  KUNG MAY ERROR PA RIN                                          │
└──────────────────────────────────────────────────────────────────┘

Gawin mo ito para makita ko ang problema:

1. Open test_print_direct.html
2. Click "Test Print Receipt"
3. Press F12 (Browser Console)
4. Screenshot ang:
   - Error message sa page
   - Console errors (F12)
5. Send sa akin yung screenshot

O kaya:

1. Open: http://localhost/caps2e2/Api/test-printer-simple.php
2. Click "📋 Ipakita ang Lahat ng Printer"
3. Screenshot kung ano ang lumabas
4. Send sa akin

╔══════════════════════════════════════════════════════════════════╗
║                    COMMON ISSUES                                 ║
╚══════════════════════════════════════════════════════════════════╝

ISSUE 1: "Access is denied"
FIX: Run XAMPP as Administrator (RUN_XAMPP_AS_ADMIN.bat)

ISSUE 2: "The system cannot find the file"
FIX: Wrong printer name. Check exact name sa Windows Settings

ISSUE 3: "Printer offline"
FIX: Turn on printer, check USB, add paper

ISSUE 4: No error but hindi nag-print
FIX: 
  - Check print queue sa Windows
  - Clear stuck jobs
  - Restart printer
  - Restart XAMPP
